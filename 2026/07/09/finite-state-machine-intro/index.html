状态机模型的介绍

发表于2026-07-09|更新于2026-07-12|计算机科学
|总字数:694|阅读时长:2分钟|浏览量:

引子

  1. 一扇自动门，它有两种状态：打开和关闭。

    • 平常是关闭的
    • 当有人靠近时，自动打开
    • 门打开后，感应人离开了，切换回关闭状态。

  2. 一个与门逻辑元器件，它输出有两种状态：高电平和低电平

    • 当输入中有低电平，则输出低电平
    • 当输入中没有低电平，则输出高电平

  3. 一个电路，它感应CPU的温度，控制风扇的转速：

    • 当CPU温度在55~65°C时，风扇转速为一档

    • 当CPU温度在66~80°C时，风扇转速为二档

    • 当CPU温度超过80°C时，风扇转速为三档

这三个例子里，都是一个系统（对象）有不同状态，当满足某些特定的条件时，系统的状态转移到另一个状态。现实里这样的例子比比皆是，特别是对于自动控制系统来说，设计者要在设计时想清楚它们的状态和切换条件。为了系统地研究这种系统，人们把它们抽象成一个理想化模型——状态机。

状态机是一种计算机领域经常用到的模型，状态机模型有很多种，这篇文章主要讲的是有限状态机（FSM）。

状态机是什么？

结合上面的例子，简单来说状态机就是描述一个系统如何在不同的状态之间切换的模型，将存在状态转换的系统抽象成一个方便理论演绎的数学模型。

状态机的四个核心要素

任何一个状态机都包含这四个部分：

  1. 状态：系统在某一时刻的稳定情况。比如自动门的“关闭”、“打开”；网络连接的“空闲”、“连接中”、“已连接”。
  2. 事件/条件：导致状态发生改变的原因。比如“有人靠近”、“超时”、“收到确认信号”。
  3. 转换：从一个状态到另一个状态的过程。它由事件触发，并可以附带条件（如“温度高于30度”）。
  4. 动作：在转换发生时或停留在某个状态时，系统执行的操作。比如“启动电机”、“显示错误信息”、“发送数据包”。

两种主要类型

根据有无输出，状态机常分为两类：

  • 摩尔型：输出动作只和当前状态有关。比如，只要处在“红灯状态”，就一直输出“停止”信号，和如何进入红灯状态无关。
  • 米利型：输出动作和当前状态以及触发事件都有关。比如，当“关门状态”下收到“有人靠近”事件，输出“开门”动作。同一个“关门状态”收到“超时”事件，可能输出“报警”动作。
